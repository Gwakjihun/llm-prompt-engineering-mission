# Example 1 (명확한 입력)
### 입력

- 로그인 기능은 OAuth2 기반으로 개발한다.
- 개발 완료 목표일은 7월 15일이다.
- QA는 개발 완료 후 진행한다.

### 출력

## 결정사항

- OAuth2 기반 로그인 기능 개발

## Action Items

| 담당 역할 | 작업 내용 | 기한 |
|------------|------------|------------|
| 개발팀 | 로그인 기능 구현 | 7월 15일 |

## 리스크

- 없음

## 후속 일정

- QA 일정 수립

## 확인 필요

- QA 시작일

# Example 2 (Action Item 중심)
### 입력

- 디자인 수정안을 다음 주까지 제출한다.
- 마케팅 초안은 기획팀에서 준비한다.

### 출력

## 결정사항

- 디자인 수정 진행

## Action Items

| 담당 역할 | 작업 내용 | 기한 |
|------------|------------|------------|
| 디자인팀 | 디자인 수정안 제출 | 다음 주 |
| 기획팀 | 마케팅 초안 작성 | 미정 |

## 리스크

- 일정 미정 항목 존재

## 확인 필요

- 마케팅 초안 제출 기한
