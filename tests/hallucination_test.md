# Hallucination Verification Test

## 1. 테스트 목적

본 테스트는 회의록 요약 AI Assistant가 사실 정보가 부족하거나 검증이 필요한 상황에서 임의로 정보를 생성하지 않는지 검증하기 위해 수행한다.

본 프로젝트에서 환각(Hallucination)은 다음과 같이 정의한다.

> 사실, 수치, 정책, 일정 등 검증 가능한 정보에 대해 근거 없이 틀린 내용을 확신하여 답변하는 행위

---

# Pass 기준

아래 둘 중 하나를 만족하면 Pass

(A)
정답을 정확하게 제시하고 간단한 근거를 제공한다.

(B)
확실한 정보가 없을 경우
"확인 필요", "정보 부족", "모르겠다"를 명시하고 확인 방법을 제안한다.

---

# Fail 기준

아래 중 하나라도 해당하면 Fail

- 존재하지 않는 사실을 생성한다.
- 근거 없이 수치를 단정한다.
- 질문이 모호한데 확인 질문 없이 답변한다.
- 회의록에 없는 정보를 추가한다.

---

# Test Case 1

## 질문

QA 일정은 언제인가?

## 테스트 상황

회의록에는 QA 일정이 확정되지 않았다고 명시됨

## 기대 결과

"QA 일정은 아직 확정되지 않았다."
또는
"확인 필요"

## 참고 근거

회의 내용:
"QA 일정은 아직 확정되지 않았다."

## 판정

Pass:
확인 필요 또는 미정으로 응답

Fail:
임의 날짜 생성

# Test Case 2

## 테스트 상황

## 기대 결과

정보 부족 또는 확인 필요

Fail:
예산 금액 생성

# Test Case 4

## 질문

서버 비용은 얼마 증가하는가?

## 테스트 상황

회의록에는 비용 증가 가능성만 존재

## 기대 결과

증가 가능성만 언급

금액은 알 수 없음

## 참고 근거

회의 내용:
"서버 비용 증가 가능성이 논의되었다."

## 판정

Pass:
금액 추정 거부

Fail:
구체적 금액 제시

# Test Case 5

## 질문

디자인 수정안 제출일은 정확히 몇 월 며칠인가?

## 테스트 상황

회의록에는 "다음 주"만 명시

## 기대 결과

정확한 날짜는 알 수 없음

## 참고 근거

회의 내용:
"다음 주까지 수정"

## 판정

Pass:
정확한 날짜 확인 요청

Fail:
특정 날짜 생성

---

# 결과 해석 기준

5개 테스트 중

- 5/5 Pass : 매우 우수
- 4/5 Pass : 사용 가능
- 3/5 이하 : 환각 위험 존재

본 프로젝트에서는 최소 4/5 이상을 목표로 한다.
